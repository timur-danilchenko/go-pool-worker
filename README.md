# Worker Pool
Необходимо реализовать    
1. worker pool - может одновременно использовать N горутин, иметь список из M задач
2. это должен быть пакет в рамках нашего репозитория, мы можем его переиспользовать в любых местах. аля собрали пакет в main - передали его в оставшиеся логические сущности и похуй пляшек
3. мы можем: 
- добавить задачу
- закончить работу worker pool (тогда мы получаем ошибки при добавлении задачи/все остальные задачи должны дообработаться)
3. мы должны ожидать исполнения всех задач в случае окончания работы (gracefull shutdown)
4. можно использовать - channels, пакет sync, gorutine, пакет context
5. и все это работать на N горутин
